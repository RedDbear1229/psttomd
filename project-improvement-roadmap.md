# Project Improvement Roadmap

## Purpose

This document consolidates the review notes created during the project audit and adds the latest findings from the full repository exploration. It should be used as the single tracking document for future improvements to the PST-to-Markdown email archive, WSL/fzf search workflow, Markdown output quality, inline image handling, and AI-assisted knowledge management.

## Target Direction

The project should behave as a WSL-first email knowledge archive:

1. Convert PST files into stable, readable Markdown.
2. Store attachments and inline images in portable archive-relative paths.
3. Build a trustworthy SQLite FTS5 index.
4. Provide fast title/body search through `mailview` and `fzf`.
5. Preserve the original email as source material.
6. Add AI features only with provenance, privacy controls, and reproducible metadata.

## Priority 1: Fix Configuration Generation

Status: New.

Issue:

- `scripts/lib/config.py::init_config_file()` writes `pst_backend` under `[archive]`.
- Runtime configuration expects `pst_backend` at the TOML top level.
- `config.example.toml` and `docs/guide.md` already document the top-level shape, so generated config files are inconsistent with the docs.

Required change:

- Move generated `pst_backend = "auto"` outside `[archive]`.
- Add a regression test for `pst2md-config init` output.

Acceptance:

- A new config generated by the CLI can control `pypff`, `readpst`, or `win32com` correctly.

## Priority 2: Normalize Attachment Paths

Status: New.

Issue:

- `scripts/lib/attachments.py::store_attachment()` stores `meta["path"]` via `str(Path)`.
- On Windows this produces paths such as `attachments\b9\file.png`.
- Markdown, WSL, Obsidian, and frontmatter consumers should use `attachments/b9/file.png`.

Required change:

```python
rel_path = dest_path.relative_to(attachments_root.parent).as_posix()
```

Acceptance:

- Frontmatter attachment paths always use `/`.
- Body image links and `mailview.get_attachments_from_md()` resolve the same file.
- `tests/test_attachments.py` passes on Windows and WSL.

## Priority 3: Keep Search Index Focused on Email Body

Status: New.

Issue:

- `scripts/build_index.py::read_body()` removes only frontmatter.
- It indexes visible Markdown sections such as title, sender summary, attachment tables, related links, and future AI blocks.
- This can pollute body search results.

Required change:

- Use `scripts/lib/md_io.py::split()` and index only `parts.body`.
- Keep subject/from/to/date/folder in their dedicated DB columns.
- Add a test where a keyword exists only in an attachment table or AI block and must not match body search.

Acceptance:

- `mailgrep --body TERM` and `Ctrl-B` in `mailview` search the original email body, not generated Markdown UI sections.

## Priority 4: Harden Markdown Output Structure

Status: Planned.

Recommended shape:

```markdown
---
msgid: "<abc@example.com>"
date: 2024-03-15T09:30:00+09:00
from: "홍길동 <hong@example.com>"
to: ["kim@example.com"]
subject: "견적서 전달드립니다"
folder: "Inbox/거래처"
thread: "t_a1b2c3d4"
attachments:
  - name: "견적서.pdf"
    size: 245760
    path: "attachments/ab/abc123.pdf"
    inline: false
---

# 견적서 전달드립니다

> [!summary]
> **보낸 사람:** 홍길동 <hong@example.com>  
> **받는 사람:** kim@example.com  
> **날짜:** 2024-03-15 09:30 +09:00  
> **폴더:** Inbox/거래처

## 첨부 파일

| 파일 | 크기 | 위치 |
|---|---:|---|
| [견적서.pdf](../../attachments/ab/abc123.pdf) | 240 KB | `attachments/ab/abc123.pdf` |

---

## 본문

원문 메일 본문

---

## 관련

- 스레드: [[t_a1b2c3d4]]
```

Required change:

- Keep frontmatter machine-readable and stable.
- Use the first `#` heading only for the subject.
- Put original source text under `## 본문`.
- Render attachments as a table.
- Preserve a clear separator so `md_io.split()` can distinguish source body from generated sections.

Acceptance:

- Markdown is readable in terminal viewers, Obsidian, VS Code, and plain text.
- Search/indexing tools can still reliably isolate the original body.

## Priority 5: Improve Inline Image Handling

Status: Planned.

Issues:

- CID matching depends too heavily on filename.
- Current regex focuses on `src="cid:..."` and can miss single quotes, whitespace, and uppercase variants.
- Inline images can appear twice: once in the body and again in the attachment section.
- Frontmatter paths and body links can diverge if one uses backslashes.

Required change:

- Capture backend metadata: `content_id`, `content_location`, `original_name`, `inline`.
- Match `cid:` by normalized `Content-ID` first, then filename fallback.
- Broaden CID regex:

```python
r'''src\s*=\s*["']cid:([^"']+)["']'''
```

- Do not render inline body images again as normal attachments.

Acceptance:

- HTML body images render in place.
- Normal attachments remain listed in the attachment table.
- Inline images are marked as `inline: true`.

## Priority 6: Improve Message Identity Stability

Status: New.

Issue:

- Missing Message-ID values are generated from sender, subject, and date.
- Multiple messages with the same sender, subject, and timestamp can collide.

Required change:

- Include folder path, PST-relative message index, attachment hash summary, or backend record identifier in the generated ID seed.
- Add a collision test with same sender/subject/date but different body.

Acceptance:

- Missing Message-ID messages do not overwrite or skip each other during conversion/resume.

## Priority 7: Strengthen `mailgrep` Query Parsing and Date Filters

Status: New.

Issues:

- Smart query parsing uses whitespace splitting, so values with spaces are fragile.
- Date filters compare ISO-like strings with timezone offsets, which can be wrong near day boundaries.

Required change:

- Use `shlex.split()` or an equivalent quoted-token parser.
- Support examples such as `folder:"Inbox/계약 문서"` and `subject:"월간 보고"`.
- Normalize message dates to UTC or add date-only columns for filtering.

Acceptance:

- Quoted smart filters work.
- `after` and `before` filters behave consistently across mixed timezone emails.

## Priority 8: Make Index Recovery Trustworthy

Status: Partially completed.

Completed:

- `mailview --doctor` reports DB/file count drift.
- `mailview` warns when Markdown files and index state diverge.

Remaining risk:

- Some detection still depends on file mtime.
- Copy/restore workflows may preserve old mtimes.

Required change:

- Prefer file count, DB path coverage, and optional manifest hashes over mtime-only detection.
- Add a command or doctor mode that reports:
  - Markdown file count;
  - DB row count;
  - missing DB paths;
  - orphan DB rows;
  - staging file status.

Acceptance:

- After backup restore or manual file copy, the user gets a clear `build-index --rebuild` recommendation.

## Priority 9: Add WSL fzf Smoke Verification

Status: New.

Issue:

- Unit tests cover command construction, but not real WSL terminal behavior with `fzf`, `change:transform(...)`, Korean input, and preview rendering.

Required change:

- Add a manual or semi-automated WSL smoke runbook:

```bash
pst2md --pst tests/data/sample.pst --out ~/mail-archive-smoke
build-index --archive ~/mail-archive-smoke --rebuild
mailgrep "견적" --archive ~/mail-archive-smoke
mailview --archive ~/mail-archive-smoke --doctor
mailview --archive ~/mail-archive-smoke
```

Manual checks:

- `Ctrl-S` searches titles across the full archive.
- `Ctrl-B` searches bodies across the full archive.
- Korean input works in the selected terminal.
- Preview does not break on image links or long subjects.

Acceptance:

- A maintainer can verify the primary WSL/fzf workflow before release.

## Priority 10: Improve fzf User Experience

Status: Partially completed.

Recommended behavior:

- Prompts should clearly show mode: `최근메일>`, `전체검색>`, `제목검색>`, `본문검색>`.
- Result rows should include match source labels: `[제목]`, `[본문]`, `[보낸이]`, `[첨부]`.
- Preview should show metadata and matching context first, full rendering only on `Enter`.
- Empty states should suggest next actions such as shorter Korean terms, title/body mode, or `build-index --rebuild`.
- Destructive actions should stay out of the primary header.

Future shortcuts:

- `Ctrl-G`: full indexed search.
- `Ctrl-S`: title search.
- `Ctrl-B`: body search.
- `Ctrl-F`: folder browser.
- `Alt-A`: has attachments.
- `Alt-R`: recently opened.
- `Ctrl-Y`: favorite toggle.

Acceptance:

- Users do not need to guess whether typing in `fzf` is local filtering or full archive search.

## Priority 11: Harden AI Features for Knowledge Management

Status: Planned.

Direction:

- AI should extend the archive into a knowledge system, not replace source search.
- Retrieval and provenance should come before chatbot-style answering.

Required principles:

- Preserve original email body.
- Store AI output separately.
- Require source links for every summary, entity, relationship, and answer.
- Store model, prompt version, input hash, generated time, and status.
- Prefer local providers by default for private archives.

Feature order:

1. Semantic search.
2. Thread-level summaries.
3. Entity extraction.
4. Related mail discovery.
5. Source-grounded Q&A.

Recommended AI block:

```markdown
<!-- AI-KNOWLEDGE:BEGIN -->
## 요약

## 결정사항

## 액션 아이템

## 관련 문서
<!-- AI-KNOWLEDGE:END -->
```

Recommended metadata:

```yaml
ai_model: "..."
ai_prompt_version: "..."
ai_input_hash: "..."
ai_updated_at: "..."
ai_status: "ok"
```

Acceptance:

- AI output can be regenerated safely.
- Every generated claim can be traced back to source emails.

## Priority 12: Control AI Cost and Privacy

Status: New.

Issues:

- `mailenrich` submits concurrent jobs before budget checks can stop all in-flight requests.
- LLM JSON parsing fails if a model returns code fences or explanatory text.
- Personal email content may be sent to external providers unless users configure local endpoints.

Required change:

- Enforce budget before submitting new work batches.
- Add robust JSON extraction around model responses.
- Add `allow_external = false` style configuration.
- Prefer environment variables for tokens.
- Add folder exclusions for sensitive areas such as `Private`, `HR`, `Legal`, `Junk`, and `Deleted Items`.
- Log metadata about sent requests without storing full sensitive prompts.

Acceptance:

- AI jobs are resumable, bounded, and explicit about external data transfer.

## Priority 13: Fix Platform-Specific Test Design

Status: New.

Issue:

- `tests/test_md_io.py` uses `chmod(0o555)` to force write errors.
- On Windows this does not reliably prevent writes.

Required change:

- Mock `Path.write_text`, `Path.replace`, or use a controlled failing wrapper.
- Keep permission-based tests only where POSIX semantics are required and mark them accordingly.

Acceptance:

- The full test suite reports meaningful failures on Windows and WSL.

## Suggested Implementation Order

1. Fix generated config shape for `pst_backend`.
2. Normalize attachment paths with `.as_posix()`.
3. Repair platform-specific tests.
4. Change index rebuild to use original body only.
5. Harden inline image CID matching and metadata.
6. Improve generated Message-ID collision resistance.
7. Upgrade smart query parsing and date filtering.
8. Add WSL fzf smoke verification.
9. Polish result rows, previews, and empty states.
10. Add AI privacy, budget, and provenance controls.
11. Implement semantic search and thread summaries.
